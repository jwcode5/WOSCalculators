WOS CALCULATOR - BUILD PLAN

PROJECT GOAL
- Build a public, alliance-friendly Whiteout Survival calculator website.
- Start with building upgrades and resource planning.
- Keep architecture modular so chief gear/charm calculators can be added later.


V1 SCOPE (BUILD THIS FIRST)
- Buildings calculator (levels start below 24).
- Current level -> target level cost calculation.
- Current level + backpack -> highest reachable level calculation.
- Construction speed and position buff fields.
- Building prerequisite auto-fill (example: Furnace target auto-fills Embassy/Research requirements).
- Custom resource chest support.
- Multi-account profile support (stored locally in browser).
- Language selector for multilingual alliance members.
- Export/share profile snapshot for alliance planning.


OUT OF SCOPE FOR V1 (V2+)
- Direct account connection to Whiteout Survival (optional, likely no official API).
- Chief gear calculator.
- Chief charm calculator.

DATA MODEL (JSON)
1) buildings.json
- Purpose: level-by-level upgrade costs and optional base construction times.
- Shape:
	{
		"version": "1.0.0",
		"buildings": {
			"furnace": {
				"displayNameKey": "building.furnace",
				"levels": {
					"1": { "wood": 1000, "meat": 800, "coal": 0, "iron": 0, "seconds": 60 },
					"2": { ... }
				}
			},
			"embassy": { ... }
		}
	}

2) prerequisites.json
- Purpose: dependency rules used by auto-fill.
- Shape:
	{
		"furnace": {
			"requires": [
				{ "building": "embassy", "offset": -1 },
				{ "building": "research_center", "offset": -1 }
			]
		}
	}

3) locales/*.json
- Purpose: UI text translations.
- Shape:
	{
		"app.title": "Whiteout Survival Calculator",
		"label.currentLevel": "Current Level",
		"label.targetLevel": "Target Level"
	}

CORE CALCULATION LOGIC
1) Validate inputs
- currentLevel >= 1
- targetLevel > currentLevel
- all numeric inputs >= 0
- selected building exists in data

2) Sum required resources
- For level = currentLevel + 1 to targetLevel:
	add buildings[building].levels[level] resources to requiredTotals

3) Apply prerequisites if enabled
- Read prerequisites rules
- For each required building:
	auto-set dependency target level using offset logic
	add dependency costs to total required costs

4) Apply time buffs
- effectiveSeconds = baseSeconds / (1 + (constructionBuff + positionBuff) / 100)

5) Compare required vs backpack
- deficit[resource] = max(required - owned, 0)
- surplus[resource] = max(owned - required, 0)

6) Chest allocation logic
- Inputs:
	chestCount and value options
- Compute total chest resource capacity
- Strategy for V1:
	auto-allocate to largest deficits first
- Output:
	how many chests needed per resource
	remaining deficits after chest use

7) Highest reachable level mode
- Start from currentLevel
- Iteratively test next level requirements
- Stop when total available resources cannot satisfy next level
- Return max reachable level and limiting resource

EXPORT/SHARE FORMAT (V1)
- Use a stable JSON schema so files stay compatible after updates.
- Suggested top-level shape:
	{
		"schemaVersion": "1.0.0",
		"exportedAt": "2026-03-21T12:34:56Z",
		"appDataVersion": "1.0.0",
		"profile": {
			"name": "PlayerName",
			"language": "en",
			"building": "furnace",
			"currentLevel": 27,
			"targetLevel": 30,
			"buffs": { "construction": 25, "position": 10 },
			"resources": { "wood": 0, "meat": 0, "coal": 0, "iron": 0 },
			"chests": { "customCount": 0, "customValue": 0 }
		}
	}
- On import:
	validate schemaVersion
	validate numeric fields >= 0
	fail gracefully with clear error if invalid
